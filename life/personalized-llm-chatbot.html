🔧 核心技术实现

1. 固件适配与烧录

基于开源 xiaozhi-esp32-main 固件进行二次开发。 ESP32 作为边缘计算节点，承担语音 I/O、Wi-Fi 通信与 LCD 驱动， 通过 HTTP/WebSocket 与云端 API 交互。固件经裁剪后烧录至开发板， 实现上电即用的独立对话终端。

2. 云端接口与 AI 引擎

在阿里云 ECS 上部署自建 API 网关，接收设备端语音数据后转发至 LLM API。服务端维护角色化的 System Prompt，包括：

  • 角色身份：姓名、背景、性格特质、说话方式
  • 语言约束：长度控制、语气风格、禁用话题
  • 关系定位：与用户的互动模式与情感基调

3. 记忆系统设计

📝 短期记忆

滑动窗口式上下文管理，保留最近 N 轮对话历史， 确保角色在多轮交互中的连贯性与一致性。

💾 长期记忆

关键信息持久化（用户偏好、历史事件）， 在对话生成时将相关内容注入 Prompt，实现跨会话的个性化交互。

4. 角色语音合成

采集角色原始配音素材作为 TTS 模型的参考音色，将 LLM 生成的文本 合成为接近角色原声的自然语音。语音输出通过 ESP32 扬声器实时播放。

5. 表情包适配

将角色表情图片转换为固件兼容的颜色格式与分辨率限制， 替换固件内置的默认表情素材，使设备 LCD 在对话中展示角色专属表情。
